# ソフトテニス分析アプリ 仕様書テンプレート（PWA対応）
## 1. 基本情報

アプリ名：ソフトテニス分析

プラットフォーム：Web/PWA（iPhone、Windowsブラウザ対応）

使用技術：HTML, CSS, JavaScript, manifest.json, service-worker.js

主な用途：ダブルス及びシングルスの試合の得点・統計管理、試合履歴保存、リアルタイム分析、主にiPhoneで利用することを考慮したUIにする

## 2. 試合設定UI
項目	詳細
味方選手	2名（内部変数：A1,A2）、ボタン表示、左側2名は味方で2名の内左から後衛と前衛
対戦相手	2名（内部変数：B1,B2）、ボタン表示、右側2名は「対戦相手」で2名の内左から後衛と前衛
味方選手と対戦相手が1名ずつしか入力されていない場合はシングルスとし、スコアボードには入力された選手名だけにする
ゲーム数	ラジオボタン：3, 5, 7, 9（デフォルトは5）
サービス権	ラジオボタン：味方 / 対戦相手（デフォルトは味方）
制御	「試合開始」押下で設定ロックし、「試合設定UI」は非表示にして「スコアボード」と「得点入力UI]を1画面に表示する
デフォルト	試合開始後は左端の味方選手が「得点入力」UIでデフォルト選択された状態にする

UIのイメージは以下の通り
味方	後衛	前衛
対戦相手	後衛	前衛
ゲーム数	3	5	7	9	（ラジオボタン）
サービス権	味方	対戦相手	（ラジオボタン）
試合開始ボタン

## 3. スコアボード
項目	詳細
ペア名	味方選手の名前を&でつなげて表示
サービス権表示	ペア名の下に赤い下線で示す
ゲームポイント	各ゲームごとに味方/対戦相手のポイントを表示
総ゲーム数	味方・対戦相手のゲーム取得数表示
ゲーム終了時	勝利セルに色付け
試合終了時	GAMEセルに色付け
表示形式	1ゲーム目から「試合設定」で選択したゲーム数分を最初から表示
現在行われているゲーム数がわかるように、該当ゲームの個所を太い罫線で囲む


UIのイメージは以下の通り（枠と罫線で表示しスコアボードのようなUIにする）
空欄	1G	2G	3G	4G	5G	GAME
A1 & A2	0	-	-	-	-	0
B1 & B2	0	-	-	-	-	0



## 4. 得点入力
前提条件1：得点については選手ごとに決定率やエラー率を算出するため、選手ごとかつ得点またはエラーをカウントする

選手選択

左2人ボタン → 味方（A1、A2）

右2人ボタン → 対戦相手（B1、B2）

選択中ボタンは緑色で表示

デフォルト選択：サービス権がある方のペアの左端の選手（A1またはB1）

得点ボタン

後衛（薄赤）：リターンエース、ストローク、ロブ

前衛（薄青）：ボレー、スマッシュ、ローボレー、ハイボレー

共通（灰色）：カット、ドロップ、ツイスト

配置：

1段目：リターンエース、ストローク、ロブ

2段目：ボレー、スマッシュ、ローボレー、ハイボレー

3段目：カット、ドロップ、ツイスト

サーバー選択時 → リターンエース非表示

レシーバー選択時 → サービスエース非表示

エラー（失点）ボタン

後衛（薄赤）：ストローク、ロブ

前衛（薄青）：ボレー、スマッシュ、ローボレー、ハイボレー

共通（灰色）：カット、ドロップ、ツイスト

「レシーブ」ボタンはレシーバー選択時のみ後衛（薄赤）のストロークの左に表示

配置は得点ボタンと同じ

サービスボタン

フォルト → ダブルフォルトに変化する（1stサーブ失敗時に変化。フォルトは失点にはならない）

ダブルフォルト → 2ndサーブ失敗として失点となる

サービスボタンの位置：エラー項目の下

フォルト未押下で他のボタン押下の場合 → 1stサービス成功
ダブルフォルト未押下の場合 → 2ndサービス成功

Undo（やり直し）ボタン

選手ボタンの右側、ドロップボタンと同じ行に配置

直前操作を取り消す（スコア・統計・ゲーム進行も戻す）

複数回やり直し可能




## 5. ゲーム進行ルール

1ゲーム：4ポイント先取（3-3の場合デュース、2点差で勝利）

ゲーム終了時：

ポイントセル色付け

サービス権交代

得点入力ボタンリフレッシュ

試合終了条件：

選択ゲーム数の過半数取得

得点入力ボタン無効化

GAMEセル色付け

サービス：サービスは最大2回打てて、1球目(1stサービス)をフォルトしても失点にはならず、2球目（2ndサービス）に失敗すると失点となる

ファイナルゲーム：味方と対戦相手のゲームポイントが設定したゲーム数/2で並んだ場合（3ゲームなら1-1、5ゲームなら2-2、7ゲームなら3-3、9ゲームなら4-4）ファイナルゲームとなる
ファイナルゲーム：7ポイント先取（6-6の場合デュース、2点差で勝利）
ファイナルゲーム：サービス権が2ポイントごとに味方と対戦相手に入れ替わる
